Cursor 源代码管理操作指南

在 Cursor 中提交代码并推送到 GitHub · 项目： ai-ad-creative-platform

1. 打开源代码管理面板

任选一种方式即可：

方式 A：左侧图标栏（最常用）
  1. 看 Cursor 最左边一竖条窄栏（活动栏）
  2. 找 分支 / 分叉 形状的图标（通常在「文件」图标下面）
  3. 点它，右侧会出现 Source Control / 源代码管理 面板

如果左侧只有文件树、看不到图标栏：

  • 菜单栏：View（查看）→ Appearance（外观）→ Activity Bar（活动栏） — 确保已勾选
  • 或 View → Open View…，搜索 Source Control 并打开
方式 B：快捷键（最快）

⌘ + Shift + G

会直接打开源代码管理面板。

方式 C：命令面板
  1. ⌘ + Shift + P 打开命令面板
  2. 输入 Source Control 或 源代码管理
  3. 选择 View: Show Source Control

2. 用面板提交并推送到 GitHub

前提：项目文件夹已在 Cursor 中打开（例如 ~/Desktop/ai-ad-creative-platform）。

  1. 改代码 — 保存文件（⌘ + S）
  2. 打开源代码管理 — ⌘ + Shift + G
  3. 看改动 — 「Changes / 更改」下列出修改过的文件；点文件名可看 diff
  4. 暂存 — 文件旁 + = 暂存单个；「Changes」旁的 + = 暂存全部
  5. 写说明 — 上方输入框写 commit message，例如：修复登录跳转问题
  6. 提交 — 点 ✓ Commit，或 ⌘ + Enter
  7. 推送 — 点 Sync Changes 或 ↑ Push；首次可能要点 Publish Branch

推送成功后，在 GitHub 仓库刷新即可看到更新。

3. 面板大概长什么样

┌─────────────────────────────────┐
│  SOURCE CONTROL                 │
├─────────────────────────────────┤
│  [ 输入 commit 说明...        ] │
│  [ ✓ Commit ]  [ ▼ ]            │
├─────────────────────────────────┤
│  Changes                        │
│    M  src/app/page.tsx      [+] │
│    M  README.md             [+] │
└─────────────────────────────────┘
         ↓ 提交后
    [ ↑ Sync Changes ]  或  [ Push ]
  • M = 已修改
  • U = 新文件
  • + = 加入本次提交（相当于 git add）

4. 常见问题

情况 处理
左侧没有 Git 图标 View → Appearance → Activity Bar 打开活动栏
Changes 是空的 确认打开的是项目根目录，且文件已保存
没有 Push / Sync 按钮 先完成 Commit；或 View → SCM 确认已启用 Git
推送要登录 已配置 SSH 密钥后一般可直接 push

5. 面板操作 vs 终端命令

面板操作 终端命令
点文件旁 + git add 文件名
Changes 旁 + git add .
写 message + ✓ Commit git commit -m "..."
Sync / Push git push

6. 日常更新 GitHub 完整流程（终端版）

cd ~/Desktop/ai-ad-creative-platform
git status
git add .
git commit -m "说明这次改了什么"
git push
提示：不要提交 .env、node_modules、.next、prisma/dev.db，这些已在 .gitignore 中排除。
